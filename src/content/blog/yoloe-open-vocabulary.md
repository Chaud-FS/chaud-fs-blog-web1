## 什么是开放词汇实例分割

传统实例分割（Mask R-CNN 一类）只能识别**训练时见过的固定类别**。换一个没见过的物体，就得重新训练或微调。**开放词汇（Open-Vocabulary）**分割允许你用一段**文本提示词**直接描述「要找什么」，模型就检测并分割出来——哪怕它从没在训练集里出现过。

**YOLOE（You Only Look Once - Efficient）** 把这套能力做到**实时**：基于 Ultralytics，支持文本提示、视觉提示、无提示三种模式，单卡即可跑出可交互的帧率。

## 为什么零样本也能 work

核心是**视觉-语言对齐**：训练阶段把「图像区域特征」和「文本词元特征」映射到同一向量空间。推理时，提示词被编码成文本向量，与图像区域向量做相似度匹配，高分区域即被分割。

> 所以「开放词汇」并非凭空理解世界，而是把「识别」转化成了「在共享空间里检索最匹配的文本」。

## 平台化时的工程实践

在 **Open-Vocabulary Instance Segmentation Platform** 里，我围绕「好用」做了几件事：

- **双形态入口**：CLI（摄像头 / 图片 / 视频流）+ PyQt5 图形界面。GUI 左侧预览、右侧实时显示掩码与标签，方便非工程同学直接用。
- **中英标签**：提示词和结果标签同时支持中英文。中文标签走一层映射表，避免模型直接输出中文时的编码问题。
- **性能监控叠层**：画面右上角实时显示 FPS、单帧推理耗时、当前显存占用，用于评估在边缘设备（如 Jetson Orin）上的可行性。

```bash
# 文本提示零样本分割
yoloe predict model=yoloe-11s-seg.pt \
  source=0 \
  texts='person, backpack, laptop, red cup' \
  show=True

# 视觉提示：给一张示例图当模板
yoloe predict model=yoloe-11s-seg.pt source=img.jpg prompts='[0.3,0.4,0.2,0.3]'
```

## 与闭集模型的对比

| 维度 | 闭集分割 | 开放词汇 (YOLOE) |
| --- | --- | --- |
| 新类别 | 需重训 / 微调 | 文本提示即可 |
| 部署灵活度 | 低 | 高 |
| 实时性 | 视模型 | YOLOE 主打实时 |
| 精度上限 | 高（特定类） | 略逊于专门微调 |

## 落地场景与边界

- 适合：**品类频繁变化**的仓储 / 产线检测、机器人**临时指定目标**、快速原型验证。
- 边界：对极度相似物体（同款不同色）或小目标，提示词要写得更具体；精度要求极高的固定场景，仍建议微调专用模型。

## 小结

开放词汇分割把「换物体就要重训模型」的重量级操作，变成了「换一句话」的轻量动作。对要快速适配真实世界的视觉系统，这是很实用的范式。
