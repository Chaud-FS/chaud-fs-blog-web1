## 什么是「具身智能链路」

具身智能让机器人在真实物理世界里**感知、思考、行动**。在 **ROS2 Nexus Embodied Assistant** 里，我搭了一条完全运行在仿真中的端到端链路：

```
Gazebo 仿真 → 视觉感知(Qwen-VL) → LLM 决策 → MoveIt2 规划 → 机械臂执行 → 反馈
```

整条链路由 `nexus_orchestrator` 节点做多模态编排，**无需任何实体硬件**即可跑通——这对前期验证和演示极其友好。

## 仿真环境：Gazebo

Gazebo 提供物理引擎与 RGB-D 相机仿真。我搭了一个桌面场景：机械臂 + 若干带颜色 / 形状的物体。仿真的价值在于**迭代零成本、可复现、不会撞坏硬件**。

## 视觉感知：Qwen-VL

相机图像送给 **Qwen-VL**，输出结构化描述而非原始像素。prompt 里我强制它返回 JSON，方便下游直接解析：

```json
{"objects":[{"label":"red_cube","position":"left-front","distance_cm":30}]}
```

## 决策大脑：LLM

LLM 把视觉、语音、运动状态统一成上下文，输出结构化任务计划：

```json
{
  "task": "把红色方块放到蓝色碗里",
  "steps": [
    {"action":"move_to","target":"red_cube"},
    {"action":"grasp"},
    {"action":"move_to","target":"blue_bowl"},
    {"action":"release"}
  ]
}
```

## 运动规划：MoveIt2

每个 `step` 由 **MoveIt2** 求解：逆运动学（IK）反解关节角、在避障约束下生成平滑轨迹、下发到仿真控制器。

## 排坑笔记

- **坐标系对齐**是最隐蔽的坑：相机系、机械臂基系、物体系必须统一（用 `tf2` 做变换），否则「看到」和「抓到」对不上。
- **规划失败要可降级**：IK 无解时，让 LLM 换抓取角度或换目标，而不是直接卡死。
- **延迟**：多模态推理慢，用**异步管线**把「感知」与「执行」解耦——感知在后台跑，执行按当前结果推进。

## 小结

具身智能不是单个模型的胜利，而是**仿真、感知、决策、规划**的协同。先把链路在仿真里跑通，是走向真实机器人最稳的第一步。
